Handling Dynamic URLs with requests

GET

How to get information from Dynamic urls like followings:
https://httpbin.org/get?page=2&count=76
https://httpbin.org/get?page=1&count=25

get_dynamic_url.py
=========================
import requests

payload = {"page": 2, "count": 25}
# resp = requests.get("http://ergast.com/api/f1/2017/circuits.json")
resp = requests.get("https://httpbin.org/get", params=payload)
print(resp.url)
print("#" * 60)
print(resp.text)

difference between json() method and json.loads()

.json() method and json.loads(). Both returned the same output. .json is a method of requests.models.Response class. It returns json body data from the request’s response.loads is a function from json package to parse string data. json.loads can be used with any string data, not only in requests context.

import requests
import json

payload = {"page": 2, "count": 25}
resp = requests.get("https://httpbin.org/get", params=payload)
print(resp.text)
print("#" * 60)
print(resp.json())
print("#" * 60)
json_resp = json.loads(resp.text)
print(json_resp)

POST

With POST request we need to send data. We can create data in dictionary format and send along with POST request. In the following example we have created an API “https://httpbin.org/post”. This API accept post requests. Notice one thing here with GET we send params, but with POST we send data, as one argument.

post_dynamic_data.py
========================
import requests

payload = {"username": "kas", "password": "kas32"}

resp = requests.post("https://httpbin.org/post", data=payload)
print(resp.url)
print(f'{"#" * 20} Printing Text {"#" * 20}')
print(resp.text)
print(f'{"#" * 20} Printing JSON {"#" * 20}')
print(resp.json())

print(f'{"#" * 20} Parse JSON {"#" * 20}')
r_dict = resp.json()
print(r_dict["form"])
print(r_dict["form"]["username"])

Timeout for every requests

We should put timeout for every requests. Else in case some request hangs or delay to respond our script will hang there forever.
In following example we have set the default delay for the api “https://httpbin.org/delay/2” as 2 secs. Now we requests the api twice. First time request time is less than 2 sec. In this case API will fail. And second case kept timeout more than 2 secs, so API will pass.

delay_test.py
================
import requests

# Set the delay for following url as 2 secs
url = "https://httpbin.org/delay/2"

global resp

try:
    # Settig timeout 2 secs or less. Request will fail
    resp = requests.post(url, timeout=2)
    print(resp.status_code)

except requests.exceptions.ReadTimeout:
    print("ReadTimeout Exception Occured")

except Exception as e:
    print(type(e))
    print("Read Timeout Occured!")

try:
    # Setting timeout more than 2 secs. Request will pass
    resp = requests.post(url, timeout=3)
    print(resp.status_code)

except requests.exceptions.ReadTimeout:
    print("ReadTimeout Exception Occured")

except Exception as e:
    print(type(e))
    print("Read Timeout Occured!")

Authentication

Basic Authentication

We have created an API “https://httpbin.org/basic-auth/kas/kas32“. Accessing this API require basic authentication. (Userid : kas & Password : kas32)

from requests.auth import HTTPBasicAuth

resp1 = requests.get("https://httpbin.org/basic-auth/kas/kas32", auth=HTTPBasicAuth("kas", "kas32"))
print(resp1.status_code)

Output
========
200

HTTP Basic Auth is so common that Requests provides a handy shorthand for using it

import requests

# Basic Authentication
resp = requests.get("https://httpbin.org/basic-auth/kas/kas32", auth=("kas", "kas32"))
print(resp.status_code)
print(resp.text)
print(resp.json())

What is netrc file

The .netrc file contains login and initialization information used by the auto-login process. It generally resides in the userâs home directory, but a location outside of the home directory can be set using the environment variable NETRC. Both locations are overridden by the command line option -N. The selected file must be a regular file, or access will be denied.

netrc Authentication

If no authentication method is given with the auth argument, Requests will attempt to get the authentication credentials for the URLâs hostname from the userâs netrc file. The netrc file overrides raw HTTP authentication headers set with headers=.
If credentials for the hostname are found, the request is sent with HTTP Basic Auth
